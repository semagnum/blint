BLint Offline

BLint Offline allows users to run BLint from the command line, fixing files if necessary.

This enables BLint for continuous integration and automated testing.

offline_blinter

Runs BLint on one or multiple blend files, applying fixes if desired.

If a file is provided, will BLint just this file. If a folder, BLint will iterate over all blend files within folder recursively.

Usage: offline_blinter.py [-h] [--fix] <blender> <path>

  • blender: path to Blender executable.

  • path: filepath. file or folder to lint

blint.offline.offline_blinter.collect_blend_files(curr_dir)

Returns all blend files in a folder.

Parameters:

curr_dir (str) – root directory to begin search for blend files.

Return type:

list[str]

blint.offline.offline_blinter.validate_args(validated_blender_path, validated_blend_path)

Validates Blender executable and blend filepaths.

Parameters:

  • validated_blender_path (str) – path to Blender executable.

  • validated_blend_path (str) – path to blend file.

Raises:

FileNotFoundError – unable to find Blender executable or blend file.

blint.offline.offline_blinter.analyze_files(blender_path, blend_path, final_blend_files, auto_fix=False, post_file_callback=<function <lambda>>)

Lints each blend file.

Parameters:

  • blender_path (str) – path to Blender executable

  • blend_path (str) – Path to blend files/folder, for printing path relative to this folder

  • final_blend_files (list[str]) – list of blend files to lint

  • auto_fix (bool) – if true, will apply any fixes available for rules

  • post_file_callback – callback after each file finishes

blinter_file_checker

Python file to run within Blender to run BLint.

If you wish to run BLint without opening Blender, use offline_blinter.py.

Parameters:

line (str) – line to print.
